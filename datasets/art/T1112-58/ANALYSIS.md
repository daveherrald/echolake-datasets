# T1112-58: Modify Registry — Allow Simultaneous Download Registry

## Technique Context

T1112 (Modify Registry) is a versatile technique used for both defense evasion and persistence. Adversaries modify registry keys to disable security tools, establish persistence mechanisms, or alter system behavior to support their objectives. The detection community focuses heavily on monitoring registry modifications to sensitive keys, particularly those affecting security controls, startup locations, and system policies.

This specific test modifies Internet Explorer settings to increase the maximum number of simultaneous connections per server (`MaxConnectionsPerServer` and `MaxConnectionsPer1_0Server`). While seemingly benign, these modifications can enable faster data exfiltration by allowing more concurrent connections to command and control infrastructure. The technique demonstrates how attackers might optimize network performance for malicious activities while flying under the radar with seemingly legitimate system tuning.

## What This Dataset Contains

This dataset captures a successful registry modification executed through PowerShell and cmd.exe. The primary evidence resides in Security event 4688 process creation events showing the complete command line execution:

The attack chain begins with PowerShell (PID 9016) spawning cmd.exe with the command:
```
"cmd.exe" /c reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings" /v "MaxConnectionsPerServer" /t REG_DWORD /d 10 /f & reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings" /v "MaxConnectionsPer1_0Server" /t REG_DWORD /d 10 /f
```

This command spawns two sequential reg.exe processes with specific registry modifications:
1. `reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings" /v "MaxConnectionsPerServer" /t REG_DWORD /d 10 /f`
2. `reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings" /v "MaxConnectionsPer1_0Server" /t REG_DWORD /d 10 /f`

Sysmon ProcessCreate events (EID 1) capture both reg.exe executions with rule matches for T1012 (Query Registry), providing the complete command lines and process relationships. The Security events show all processes exiting with status 0x0 (success), confirming the registry modifications completed successfully.

## What This Dataset Does Not Contain

Notably absent are Sysmon registry modification events (EID 13), which would directly capture the registry value changes. This gap likely stems from the sysmon-modular configuration not monitoring the specific registry path `HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings`. The PowerShell channel contains only test framework boilerplate (Set-StrictMode, Set-ExecutionPolicy Bypass) rather than the actual technique execution commands.

Windows Defender did not block this activity, as evidenced by the successful process executions and zero exit codes. No network connections are generated by this technique, so DNS or network connection events are appropriately absent.

## Assessment

This dataset provides strong process-level telemetry for detecting registry modification techniques executed via command-line tools. The Security 4688 events with full command-line logging offer excellent detection opportunities, clearly showing suspicious reg.exe usage patterns. The Sysmon ProcessCreate events add valuable context with parent-child process relationships and technique tagging.

However, the absence of actual registry modification telemetry (Sysmon EID 13) limits the dataset's utility for building detections based on specific registry changes rather than process behavior. For a complete registry modification detection strategy, you would need additional configuration to monitor the affected registry paths or rely entirely on process-based detection logic.

## Detection Opportunities Present in This Data

1. **Command-line analysis for reg.exe with Internet Settings modifications** - Security 4688 events show reg.exe modifying `HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings` with MaxConnections* values, indicating potential network optimization for malicious purposes

2. **Process chain analysis for PowerShell spawning cmd.exe spawning reg.exe** - Sysmon EID 1 events reveal suspicious process lineage: powershell.exe → cmd.exe → reg.exe, particularly when modifying system configuration

3. **Bulk registry modification detection** - Security events show multiple sequential reg.exe executions within seconds, suggesting scripted registry changes rather than manual administration

4. **Internet Explorer configuration tampering** - The specific registry path and values (MaxConnectionsPerServer, MaxConnectionsPer1_0Server) indicate attempts to modify browser networking behavior

5. **SYSTEM context registry modifications** - All events show execution under NT AUTHORITY\SYSTEM, making registry changes that affect user browsing behavior while running with elevated privileges
