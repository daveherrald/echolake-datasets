# T1069.001-5: Local Groups — Wmic Group Discovery

## Technique Context

T1069.001 (Local Groups) is a discovery technique where attackers enumerate local group memberships to understand user privileges, administrative access, and potential lateral movement opportunities. This technique is commonly used during initial reconnaissance and privilege escalation phases to map the security landscape of a compromised system.

The detection community focuses on identifying suspicious group enumeration activities, particularly when performed by unexpected processes, at unusual times, or in rapid succession. WMIC (Windows Management Instrumentation Command-line) is a frequently observed tool for this technique due to its built-in capabilities and living-off-the-land status, making it an attractive option for attackers seeking to avoid detection while gathering system information.

## What This Dataset Contains

This dataset captures a straightforward WMIC-based local group discovery execution with comprehensive telemetry across multiple channels:

**Process Chain**: The complete execution flow is documented in Security event 4688 logs, showing PowerShell spawning `cmd.exe /c wmic group get name`, which then launches `WMIC.exe group get name`.

**Command Line Evidence**: The exact command `"cmd.exe" /c wmic group get name` and `wmic group get name` are preserved in Security logs with full command-line auditing enabled.

**Sysmon Process Creation**: Events show the creation of both cmd.exe (EID 1) and whoami.exe processes with detailed metadata including hashes, parent process relationships, and integrity levels. The cmd.exe process shows `CommandLine: "cmd.exe" /c wmic group get name` with parent process PowerShell.

**WMI Activity Indicators**: Sysmon EID 7 events capture WMIC loading WMI-related DLLs including `wmiutils.dll`, providing forensic evidence of WMI subsystem engagement for the group enumeration query.

**Privilege Escalation Telemetry**: Security EID 4703 events document token privilege adjustments for the WMIC process, including elevated privileges like `SeSecurityPrivilege`, `SeTakeOwnershipPrivilege`, and `SeBackupPrivilege`.

## What This Dataset Does Not Contain

The dataset lacks several elements that would provide a complete operational picture:

**WMI Query Results**: The actual output of the group enumeration (which groups exist on the system) is not captured in the logs, as Windows event logs don't typically record command output.

**Network Activity**: No network connections are generated by this local enumeration technique, so network-based detection opportunities are absent.

**Registry Access**: The WMIC group query doesn't generate observable registry access events, limiting registry-based detection approaches.

**File System Artifacts**: Beyond PowerShell profile creation, no significant file system activity occurs that would provide additional forensic evidence.

## Assessment

This dataset provides excellent coverage for detecting WMIC-based group discovery activities. The combination of Security 4688 process creation events with full command-line logging and Sysmon process creation events offers multiple high-fidelity detection opportunities. The inclusion of WMI-specific DLL loading events (wmiutils.dll) provides additional confirmation of WMI subsystem usage.

The privilege adjustment events (4703) are particularly valuable as they show the elevated privileges required for group enumeration, which can help distinguish legitimate administrative activities from potentially malicious reconnaissance. The complete process ancestry chain from PowerShell through cmd.exe to WMIC provides clear evidence of the execution path.

For detection engineering, this dataset demonstrates how living-off-the-land techniques generate distinctive telemetry patterns that can be reliably identified without requiring specialized endpoint detection tools.

## Detection Opportunities Present in This Data

1. **WMIC Group Enumeration Command**: Detect `wmic.exe` execution with command lines containing `group get` or similar group enumeration syntax via Security 4688 or Sysmon EID 1 events.

2. **Unusual WMIC Parent Processes**: Alert on WMIC processes spawned by non-administrative tools like cmd.exe or PowerShell in unexpected contexts, using parent process relationships in Sysmon data.

3. **WMI Subsystem DLL Loading**: Monitor for processes loading `wmiutils.dll` outside of expected system administration contexts using Sysmon EID 7 events.

4. **Privilege Escalation for Discovery**: Detect Security 4703 token privilege adjustment events where elevated privileges (SeSecurityPrivilege, SeTakeOwnershipPrivilege) are granted to discovery-related processes.

5. **Rapid Discovery Tool Execution**: Correlate WMIC group queries with other reconnaissance commands (like whoami.exe execution shown in this dataset) to identify broader discovery campaigns.

6. **PowerShell-to-WMIC Execution Chain**: Monitor for PowerShell processes spawning cmd.exe that subsequently launches WMIC with group-related parameters, indicating potential scripted reconnaissance activities.
