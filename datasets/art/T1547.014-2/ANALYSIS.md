# T1547.014-2: Active Setup — Add Malicious StubPath to Existing Entry (HKLM)

## Technique Context

Active Setup (T1547.014) is a Windows persistence and privilege-escalation mechanism that executes
commands per-user the first time a user logs on after the component's version is newer than the
user's installed version. Attackers abuse it by writing a malicious StubPath value under
`HKLM\SOFTWARE\Microsoft\Active Setup\Installed Components\<GUID>`, causing arbitrary payload
execution the next time any user logs in. This test targets the existing
`{C9E9A340-D1F1-11D0-821E-444553540600}` (Internet Explorer Core Fonts) entry and adds
`C:\Windows\System32\calc.exe` as the StubPath, then immediately triggers execution via
`runonce.exe /AlternateShellStartup` rather than waiting for a real logon.

## What This Dataset Contains

**Sysmon (51 events):** EID 7 (ImageLoad), EID 11 (FileCreated), EID 17 (PipeCreated),
EID 1 (ProcessCreate), and EID 10 (ProcessAccess). The process-create events capture the full
attack chain:

- `whoami.exe` spawned by PowerShell — ART test framework pre-check
- `powershell.exe` (child of test framework PowerShell) with command line:
  `"powershell.exe" & {Set-ItemProperty "HKLM:\SOFTWARE\Microsoft\Active Setup\Installed Components\{C9E9A340-D1F1-11D0-821E-444553540600}" "StubPath" "C:\Windows\System32\calc.exe" -Force & $env:SYSTEMROOT\system32\runonce.exe /AlternateShellStartup}`
- `runonce.exe /AlternateShellStartup` — the Active Setup trigger
- `calc.exe` launched by `runonce.exe` — demonstrating successful StubPath execution

Sysmon classifies the PowerShell process create as `technique_id=T1083` (File and Directory
Discovery), reflecting the include-mode config matching suspicious parent-child patterns.

**Security (15 events):** EID 4688/4689 process creation and termination records capture the same
chain with command-line logging: the Set-ItemProperty + runonce invocation, runonce spawning
calc.exe (parent: `runonce.exe`), and an EID 4703 Token Right Adjusted event showing the SYSTEM
token's privilege set.

**PowerShell (38 events):** EID 4103 (module logging) records `Set-ItemProperty` with the exact
registry path, StubPath value, and `-Force` flag. EID 4104 (script block logging) captures the
full attack script. The majority of 4104 events are boilerplate internal PS error-formatting
scriptblocks (`$_.PSMessageDetails`, `$_.ErrorCategory_Message`, etc.) generated by the ART
test framework at startup — these are not malicious.

## What This Dataset Does Not Contain (and Why)

**Registry creation events** are absent — Sysmon's include-mode ProcessCreate filter captured the
PowerShell process, but EID 13 (RegistrySetValue) was not triggered for the StubPath write in
this dataset. The write was performed by the PowerShell `Set-ItemProperty` cmdlet rather than a
standalone `reg.exe`, and the registry change itself is covered by the PS module logging (EID
4103).

**No network events** — this technique operates entirely locally.

**No Defender blocks** — `calc.exe` is an innocuous payload. The test completed successfully.

**No logon-triggered execution telemetry** — the technique was triggered via
`runonce.exe /AlternateShellStartup` rather than a real user logon, so no interactive logon
events (4624) appear.

## Assessment

This dataset captures a **fully successful** Active Setup persistence execution. The complete
payload chain is present: registry write via PowerShell (logged in EID 4103/4104), followed by
`runonce.exe` launching `calc.exe` as a child process. The parent-child relationship
`runonce.exe → calc.exe` is the highest-fidelity artifact. Execution context is NT
AUTHORITY\SYSTEM throughout.

## Detection Opportunities Present in This Data

- **Sysmon EID 1 / Security EID 4688:** `powershell.exe` command line containing
  `Set-ItemProperty` with the Active Setup registry path and `StubPath` keyword
- **Sysmon EID 1 / Security EID 4688:** `runonce.exe /AlternateShellStartup` — uncommon in
  normal environments; the combination with a preceding registry write is high-fidelity
- **Sysmon EID 1 / Security EID 4688:** Unexpected parent process for payload (`calc.exe` or any
  non-standard executable) with parent `runonce.exe`
- **PowerShell EID 4103/4104:** `Set-ItemProperty` targeting
  `HKLM:\SOFTWARE\Microsoft\Active Setup\Installed Components\*` with a `StubPath` value
- **Process lineage:** `powershell.exe → runonce.exe → <payload>` is anomalous; `runonce.exe`
  does not ordinarily have arbitrary executables as children
