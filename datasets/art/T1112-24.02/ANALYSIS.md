# T1112-24: Modify Registry — Activate Windows NoSetTaskbar Group Policy Feature

## Technique Context

The `NoSetTaskbar` Group Policy value (set to `1` in `HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer`) prevents users from modifying taskbar settings — pinning and unpinning programs, changing taskbar location, configuring notification area icons. While this is a lower-impact restriction than NoControlPanel or NoRun, it fits the same pattern of UI-restriction modifications that collectively degrade a victim's ability to reconfigure their environment in response to an attack.

The NoSetTaskbar setting is more commonly deployed by IT administrators in managed enterprise environments (kiosk deployments, lab machines) than by attackers, which makes it a somewhat lower-fidelity indicator in isolation. However, in the context of this series of modifications — where multiple Explorer policy values are set in sequence during the same session — the pattern of systematic UI restriction is the meaningful behavioral signal.

This test stands out from T1112-19, T1112-21, and T1112-22 for one reason: the Security channel contains 29 events rather than the typical 4–5. The additional events are Security EID 4798 (user local group membership enumeration) and EID 4799 (security-enabled local group membership enumeration) generated by `WmiPrvSE.exe` and `cribl.exe` respectively — genuine background system activity captured in the 5-second collection window.

## What This Dataset Contains

This dataset captures 141 events across three channels (93 PowerShell, 29 Security, 19 Sysmon) collected over a 5-second window (2026-03-14T23:49:33Z–23:49:38Z) on ACME-WS06 with Defender disabled.

**Process Creation Chain (Security EID 4688):**

Five EID 4688 events:
1. `whoami.exe` — pre-test identity check
2. `cmd.exe` with command: `"cmd.exe" /c reg add "HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer" /v NoSetTaskbar /t REG_DWORD /d 1 /f`
3. `reg.exe` with command: `reg  add "HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer" /v NoSetTaskbar /t REG_DWORD /d 1 /f`
4. `whoami.exe` — post-test identity check
5. `C:\Windows\system32\wbem\wmiprvse.exe -Embedding` — WMI Provider Host spawned by WMI service, unrelated to the attack

**Security EID 4798 — User Local Group Membership Enumeration:**

Five EID 4798 events show `WmiPrvSE.exe` (PID 0x1298) enumerating local group membership for individual accounts:
- `DefaultAccount` (SID `S-1-5-21-...-503`)
- `Administrator` (SID `S-1-5-21-...-500`)
- `mm11711` (SID `S-1-5-21-...-1000`)
- `WDAGUtilityAccount` (SID `S-1-5-21-...-504`)
- `Guest` (SID `S-1-5-21-...-501`)

This is WMI-based account enumeration, consistent with a management agent or monitoring tool performing local account discovery. The account SIDs confirm the local accounts present on ACME-WS06: the standard `Administrator`, `Guest`, `DefaultAccount`, and `WDAGUtilityAccount` built-in accounts plus the `mm11711` domain user account. This is not attack-related activity.

## What This Dataset Does Not Contain

- **Sysmon EID 13 (Registry Value Set):** Direct registry modification events are not captured for this path.
- **The source of the WMI group enumeration:** The 4798 events show WmiPrvSE.exe performing the enumeration, but the WMI consumer (the application that triggered WmiPrvSE.exe to run) is not visible in process creation telemetry because WmiPrvSE.exe is a shared hosting process.
- **Cribl Edge's full account inventory schedule:** The 4799 events show one inventory cycle. Without the Cribl configuration, the periodicity and trigger conditions for this enumeration are not visible from the Windows events alone.

## Assessment

This dataset is the most event-rich in the T1112 series primarily because of the coincident Cribl Edge group enumeration and WMI account discovery. The Security channel's 29 events (vs. 4–5 in other T1112 tests) are almost entirely background system activity. An analyst encountering this event set would need to correctly attribute the EID 4799 activity to Cribl Edge (by process image path) and the EID 4798 activity to WmiPrvSE.exe (and then trace back to the WMI consumer) before concluding whether the account enumeration is attack-related.

The EID 4799 events from Cribl Edge are a real-world example of a monitoring agent generating telemetry that is structurally identical to T1087 (Account Discovery) activity. Cribl's enumeration hits every local group methodically — exactly as a post-exploitation reconnaissance script would. This dataset provides a concrete example of why process image path attribution matters for triage of account enumeration events.

Compared to the defended variant (83 events: 34 PowerShell, 12 Security, 37 Sysmon), this undefended dataset (141 events) is substantially larger, but the difference is dominated by the ambient EID 4798/4799 activity, not by the T1112 technique itself.

## Detection Opportunities Present in This Data

**EID 4688 / Sysmon EID 1 — NoSetTaskbar in reg add Command:**
The `reg add` command targeting the Explorer Policies key with value `NoSetTaskbar` is the direct attack indicator. Pattern coverage for any value add under `HKCU\...\Policies\Explorer` applies here as in all other T1112 tests in this series.

**Security EID 4799 — Cribl Edge Group Enumeration Baseline:**
The EID 4799 events from `C:\Program Files\Cribl\bin\cribl.exe` enumerating local groups provide a baseline for what this agent's normal account inventory activity looks like. In a detection context, `cribl.exe` generating EID 4799 events should be allowlisted after verification; the same pattern from an unexpected process image (e.g., `powershell.exe`, `cmd.exe`, or a binary in `C:\Users\...`) would be an indicator of account reconnaissance.

**Security EID 4798 — WMI Account Enumeration Baseline:**
Five EID 4798 events from `WmiPrvSE.exe` enumerating local account membership across all local accounts is consistent with a WMI-based management tool performing account inventory. This pattern is benign here but represents the same telemetry signature that WMI-based account discovery (T1087.001) would generate.

**Combined EID 4798+4799 Pattern from Unexpected Process:**
If either EID 4798 or 4799 events appear with a process image other than known management agents (WmiPrvSE.exe, cribl.exe, or similar), the systematic enumeration of all local accounts and groups in a short time window is a high-fidelity indicator of post-exploitation account reconnaissance.
