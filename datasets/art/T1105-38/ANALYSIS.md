# T1105-38: Ingress Tool Transfer — Download a file with OneDrive Standalone Updater

## Technique Context

T1105 (Ingress Tool Transfer) represents a critical capability where adversaries transfer tools or files from an external system into a compromised environment. The OneDrive Standalone Updater technique is particularly insidious because it abuses a legitimate Microsoft application to download arbitrary content from external URLs. This approach leverages the principle of "living off the land" - using trusted system binaries to bypass security controls.

The OneDrive Standalone Updater (OneDriveStandaloneUpdater.exe) is designed to check for and download OneDrive updates. By manipulating registry keys in `HKCU:\Software\Microsoft\OneDrive\UpdateOfficeConfig`, attackers can redirect the updater to fetch content from attacker-controlled URLs. This technique is attractive to adversaries because the network requests originate from a signed Microsoft binary, potentially evading application allowlisting and network monitoring focused on unsigned executables.

Detection engineers typically focus on registry modifications to OneDrive update configuration keys, unexpected network connections from OneDriveStandaloneUpdater.exe, and the process being launched outside of normal update contexts.

## What This Dataset Contains

This dataset captures a failed attempt at the OneDrive Standalone Updater technique. The Security logs show PowerShell process creation with the full command line containing the attack script (EID 4688, process 15836). The PowerShell channel captures extensive script block logging showing the test attempting to check for `OneDriveStandaloneUpdater.exe` at `C:\Users\ACME-WS02$\AppData\Local\Microsoft\OneDrive\OneDriveStandaloneUpdater.exe` (EID 4103: `Test-Path` invocation).

The PowerShell execution reveals the intended attack flow: creating registry keys under `HKCU:\Software\Microsoft\OneDrive\UpdateOfficeConfig` and setting malicious URLs pointing to `https://raw.githubusercontent.com/redcanaryco/atomic-red-team/master/LICENSE.txt` for properties like `UpdateRingSettingURLFromOC`, `ODSUUpdateXMLUrlFromOC`, and `UpdateXMLUrlFromOC`.

Sysmon captures the complete PowerShell process chain (EID 1) with ProcessGUID `{9dc7570a-696a-69b4-6d44-000000001000}` and the full command line showing the registry manipulation commands. The test ultimately fails because OneDriveStandaloneUpdater.exe is not present on the system, with PowerShell logging a `Write-Host` command stating "OneDriveStandaloneUpdater.exe not found" (EID 4103).

## What This Dataset Does Not Contain

The dataset lacks the critical telemetry that would be generated by a successful execution. Most importantly, there are no Sysmon EID 1 (Process Create) events for OneDriveStandaloneUpdater.exe because the binary doesn't exist on the test system. Consequently, there are no Sysmon EID 3 (Network Connection) events showing the standalone updater making HTTP requests to the malicious URLs.

The dataset also lacks Sysmon EID 13 (Registry Value Set) events because the sysmon-modular configuration doesn't monitor the specific OneDrive registry paths being targeted. Without these registry modification events, detection engineers lose visibility into the configuration changes that enable this technique.

Additionally, there are no file creation events (beyond PowerShell startup profiles) showing downloaded payloads, and no subsequent process execution that would occur if the technique had successfully retrieved and executed additional tools.

## Assessment

This dataset provides limited utility for detection engineering of the OneDrive Standalone Updater technique. While it captures the preparation phase (PowerShell script execution and registry manipulation commands), the core technique behavior - the actual abuse of OneDriveStandaloneUpdater.exe to download external content - is not demonstrated.

The most valuable telemetry present is the PowerShell command-line logging showing the complete attack script, which enables detection of the preparation phase. However, for comprehensive detection development, analysts would need additional datasets showing successful executions with OneDriveStandaloneUpdater.exe present, registry monitoring enabled, and network connections captured.

The dataset's strength lies in demonstrating how Atomic Red Team tests gracefully handle missing dependencies, providing insight into attack prerequisites and failure modes that defenders should understand.

## Detection Opportunities Present in This Data

1. **PowerShell command-line detection** - Security EID 4688 and Sysmon EID 1 capture the full PowerShell command containing OneDrive registry manipulation and references to UpdateOfficeConfig registry paths

2. **Script block analysis** - PowerShell EID 4104 events contain the complete attack script, enabling detection of registry modification commands targeting OneDrive update configuration

3. **Test-Path enumeration** - PowerShell EID 4103 shows reconnaissance for OneDriveStandaloneUpdater.exe, which could indicate preparation for this technique

4. **Suspicious PowerShell cmdlet sequences** - The combination of `New-Item`, `Set-ItemProperty` targeting OneDrive registry paths with external URLs is a strong indicator

5. **Process ancestry analysis** - The parent-child relationship between PowerShell processes could identify script-driven attacks targeting OneDrive components

6. **Registry path targeting** - References to `HKCU:\Software\Microsoft\OneDrive\UpdateOfficeConfig` in command lines or script blocks indicate potential OneDrive abuse
