# T1053.005-9: Scheduled Task — PowerShell Modify A Scheduled Task

## Technique Context

T1053.005 (Scheduled Task/Job: Scheduled Task) is a fundamental persistence and execution technique where attackers create, modify, or abuse Windows Task Scheduler tasks to maintain access or execute code at specific intervals or events. This technique is particularly valuable for attackers because scheduled tasks can execute with elevated privileges, survive reboots, and blend in with legitimate system administration activities.

The detection community focuses heavily on task creation events, command-line analysis for suspicious executables, and monitoring registry modifications to the Task Scheduler cache. PowerShell-based task manipulation is especially concerning because it provides a native, "living off the land" approach that doesn't require dropping additional tools. The modification of existing tasks (as demonstrated in this test) is particularly stealthy since it leverages an already-present task structure.

## What This Dataset Contains

This dataset captures a complete PowerShell-based scheduled task creation and modification sequence. The attack creates a task named "AtomicTaskModifed" that initially executes `cmd.exe`, then modifies it to execute `notepad.exe` instead.

**Process Execution Chain:**
- Security EID 4688: PowerShell child process with full command line: `"powershell.exe" & {$Action = New-ScheduledTaskAction -Execute \"cmd.exe\"\n$Trigger = New-ScheduledTaskTrigger -AtLogon\n$User = New-ScheduledTaskPrincipal -GroupId \"BUILTIN\Administrators\" -RunLevel Highest\n$Set = New-ScheduledTaskSettingsSet\n$object = New-ScheduledTask -Action $Action -Principal $User -Trigger $Trigger -Settings $Set\nRegister-ScheduledTask AtomicTaskModifed -InputObject $object\n$NewAction = New-ScheduledTaskAction -Execute \"Notepad.exe\"\nSet-ScheduledTask \"AtomicTaskModifed\" -Action $NewAction}`

**PowerShell Telemetry:**
- PowerShell EID 4103: Command invocations for `New-ScheduledTaskAction` (both `cmd.exe` and `notepad.exe`), `New-ScheduledTaskTrigger -AtLogon`, `New-ScheduledTaskPrincipal -GroupId "BUILTIN\Administrators" -RunLevel Highest`, `Register-ScheduledTask`, and `Set-ScheduledTask`
- PowerShell EID 4104: Full script block containing the complete task creation and modification logic

**Registry Activity:**
- Sysmon EID 13: Registry writes to `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\TaskCache\Tree\AtomicTaskModifed\Index`, `SD`, and `Id` values

**File System Activity:**
- Sysmon EID 11: Task file creation at `C:\Windows\System32\Tasks\AtomicTaskModifed`

**Task Scheduler Events:**
- TaskScheduler EID 106: Task registration for "\AtomicTaskModifed"
- TaskScheduler EID 140: Task update for "\AtomicTaskModifed"

## What This Dataset Does Not Contain

The dataset is missing some expected telemetry that would strengthen detection coverage:

- **Sysmon ProcessCreate events**: The PowerShell processes are missing from Sysmon EID 1 logs due to the include-mode filtering in the sysmon-modular configuration, though the child `whoami.exe` process is captured
- **Task execution events**: The created task doesn't actually execute during the test window, so there are no TaskScheduler execution events (EID 200, 201)
- **XML content inspection**: While the task file is created, the dataset doesn't include the actual XML content that would reveal the task configuration details
- **Network activity**: No network connections are generated by this technique

## Assessment

This dataset provides excellent coverage for detecting PowerShell-based scheduled task manipulation. The combination of PowerShell script block logging (EID 4104), command invocation logging (EID 4103), and Security process creation events (EID 4688) creates multiple detection opportunities. The registry modifications captured by Sysmon EID 13 provide additional forensic value, and the native TaskScheduler events offer clean, purpose-built telemetry.

The PowerShell telemetry is particularly strong, capturing not just the execution but the full parameter binding for each task-related cmdlet. This level of detail enables precise detection rules that can identify both the initial task creation and the subsequent modification. The combination of Security and TaskScheduler events provides redundant coverage that increases detection reliability.

## Detection Opportunities Present in This Data

1. **PowerShell Script Block Analysis** - PowerShell EID 4104 containing `Register-ScheduledTask` and `Set-ScheduledTask` cmdlets in the same script block, indicating task creation followed by immediate modification

2. **Suspicious Task Principal Configuration** - PowerShell EID 4103 showing `New-ScheduledTaskPrincipal` with `GroupId="BUILTIN\Administrators"` and `RunLevel=Highest`, indicating privilege escalation intent

3. **Task Registration and Modification Sequence** - TaskScheduler EIDs 106 and 140 occurring within a short time window for the same task name, indicating rapid task manipulation

4. **Registry Cache Manipulation** - Sysmon EID 13 events writing to `Schedule\TaskCache\Tree\<TaskName>\*` registry keys, particularly when correlated with PowerShell activity

5. **High-Privilege PowerShell Command Lines** - Security EID 4688 with command lines containing both `Register-ScheduledTask` and `Set-ScheduledTask` in the same execution context

6. **Task File Creation Correlation** - Sysmon EID 11 showing task file creation in `C:\Windows\System32\Tasks\` correlated with PowerShell script execution and TaskScheduler events

7. **PowerShell Cmdlet Parameter Analysis** - PowerShell EID 4103 events showing task action changes from one executable (`cmd.exe`) to another (`notepad.exe`) within the same PowerShell session
