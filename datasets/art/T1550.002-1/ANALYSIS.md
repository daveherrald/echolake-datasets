# T1550.002-1: Pass the Hash — Mimikatz Pass the Hash

## Technique Context

Pass the Hash (T1550.002) allows an attacker to authenticate to remote services using a captured NTLM hash without knowing the plaintext password. Mimikatz implements this via `sekurlsa::pth`, which creates a new process running under a token forged with the supplied hash. This is a foundational lateral movement technique in Windows environments and remains effective against NTLM-reliant services even on fully patched systems.

## What This Dataset Contains

The dataset captures the full execution attempt of Mimikatz `sekurlsa::pth` launched from PowerShell under `NT AUTHORITY\SYSTEM`. The attack command is recorded verbatim in Security event 4688:

> `"cmd.exe" /c %tmp%\mimikatz\x64\mimikatz.exe "sekurlsa::pth /user:Administrator /domain:%userdnsdomain% /ntlm:cc36cf7a8514893efccd3324464tkg1a"`

The cmd.exe process exits immediately with status `0xC0000022` (STATUS_ACCESS_DENIED), confirming Windows Defender blocked the mimikatz binary before it could execute. Sysmon event 8 (CreateRemoteThread) records PowerShell creating a remote thread into an unknown target process, and event 10 (ProcessAccess with `GrantedAccess: 0x1FFFFF`) records PowerShell opening `whoami.exe` with full access — both consistent with the ART test framework probing child process activity. A `whoami.exe` process creation is also logged in both Sysmon EID 1 and Security 4688; this is the ART pre-flight identity check, not Mimikatz output.

The PowerShell log contains 42 events dominated by test framework boilerplate: repeated `Set-StrictMode`, `PSMessageDetails`, `ErrorCategory_Message`, and `OriginInfo` script blocks generated by the ART framework's error-handling infrastructure, plus a `Set-ExecutionPolicy -Scope Process -Force -ExecutionPolicy Bypass` invocation. No Mimikatz-specific script blocks appear in PowerShell logs because the tool was invoked as a native binary from cmd.exe, not via PowerShell directly.

Sysmon image-load events (EID 7) show PowerShell loading `System.Management.Automation.ni.dll` (tagged `technique_id=T1059.001`), Defender's `MpOAV.dll` and `MpClient.dll` (tagged `technique_id=T1574.002`), and .NET CLR DLLs (`clr.dll`, `clrjit.dll`, `mscorlib.ni.dll`) tagged `technique_id=T1055`. A network connection from `MsMpEng.exe` to `172.178.160.26:443` (Sysmon EID 3, tagged `technique_id=T1036`) reflects Defender's cloud telemetry upload triggered by the detection.

## What This Dataset Does Not Contain (and Why)

There is no successful NTLM token injection, no `sekurlsa::pth` completion, and no spawned process running under the stolen hash. Defender blocked the mimikatz executable at launch (`0xC0000022`), so no post-exploitation activity occurred. There are no Kerberos or logon events (4624/4625/4648) because no actual authentication was attempted. There is no LSASS access (Sysmon EID 10 against `lsass.exe`) because Mimikatz never reached its credential dumping stage. The Sysmon ProcessCreate filter (include-mode, LOLBin-focused) did not fire for mimikatz itself since it is not in the include list — the cmd.exe that launched it was captured via Security 4688 instead.

## Assessment

This dataset documents a Defender-blocked Mimikatz Pass the Hash attempt. The primary detection value is in Security 4688 capturing the exact `sekurlsa::pth` command line with a syntactically correct NTLM hash argument, combined with an immediate `0xC0000022` exit of the launching cmd.exe. The Sysmon EID 8 CreateRemoteThread and EID 10 ProcessAccess events provide secondary behavioral signals. The dataset accurately represents what telemetry an organization with active Defender would see: the attempt is visible, but success telemetry is absent.

## Detection Opportunities Present in This Data

- **Security 4688 command-line content**: The string `sekurlsa::pth` and `/ntlm:` with a 32-character hex hash in a cmd.exe command line is unambiguous. This is the highest-confidence detection in the dataset.
- **Security 4689 exit status 0xC0000022**: cmd.exe or any process exiting with STATUS_ACCESS_DENIED within milliseconds of launching a known-malicious binary path (`%tmp%\mimikatz`) is a reliable block indicator.
- **Sysmon EID 8 CreateRemoteThread from powershell.exe**: PowerShell creating remote threads into child processes is anomalous and warrants investigation regardless of the target binary.
- **Sysmon EID 10 GrantedAccess 0x1FFFFF from powershell.exe**: Full process access from a PowerShell process to a child is a high-confidence behavioral signal.
- **Parent-child relationship**: `powershell.exe` spawning `cmd.exe` which attempts to execute a binary from `%TEMP%\mimikatz\` is a detectable process tree pattern.
