# T1046-3: Network Service Discovery — Port Scan NMap for Windows

## Technique Context

Network Service Discovery (T1046) is a Discovery tactic technique where adversaries attempt to identify network services running on systems within their environment. Port scanning is one of the most fundamental methods for this technique, allowing attackers to enumerate open ports and identify potentially exploitable services. Nmap is the de facto standard port scanning tool, and while primarily associated with Linux environments, Windows ports exist and are increasingly used in post-exploitation scenarios.

From a detection perspective, the security community focuses on identifying port scanning behavior through network-based detection (rapid connection attempts to multiple ports/hosts) and host-based indicators (execution of scanning tools, unusual network connections from processes). This technique is particularly important because it often represents the reconnaissance phase preceding lateral movement or service exploitation.

## What This Dataset Contains

This dataset captures a PowerShell-executed nmap scan targeting localhost (127.0.0.1). The key evidence includes:

**Process Chain:** The attack begins with PowerShell process PID 6868, which spawns a child PowerShell process PID 7176 with command line `"powershell.exe" & {nmap 127.0.0.1}`. This child process was created to execute the nmap command against the localhost address.

**PowerShell Script Blocks:** The PowerShell channel captured the exact command execution in EID 4104 script block events, showing `& {nmap 127.0.0.1}` and the inner `{nmap 127.0.0.1}` block (ScriptBlock IDs 507b7b11-26c2-43f3-ba3c-5349a49fad51 and 74724459-0b7a-40a1-b0e3-cb17aeb6a90f).

**Security Events:** Security EID 4688 process creation events show the PowerShell process creation with the full command line `"powershell.exe" & {nmap 127.0.0.1}`, providing command-line audit trail.

**Process Access:** Sysmon EID 10 events show the PowerShell process accessing both the whoami.exe process (PID 5744) and the child PowerShell process (PID 7176) with full access rights (0x1FFFFF), indicating normal process management operations.

**Typical PowerShell Telemetry:** The dataset includes standard PowerShell initialization artifacts including .NET runtime loading, named pipe creation for PowerShell hosts, and execution policy bypass operations.

## What This Dataset Does Not Contain

**No Nmap Process Execution:** Critically, there are no Sysmon EID 1 process creation events for nmap.exe itself, nor are there Security EID 4688 events for nmap execution. This suggests that either nmap was not installed on the system, the command failed, or Windows Defender blocked the execution.

**No Network Connections:** There are no Sysmon EID 3 NetworkConnect events showing the actual port scanning network traffic that would be generated by a successful nmap execution.

**No DNS Query Events:** Missing Sysmon EID 22 DNS query events that might occur during scanning operations.

**No File Access for Nmap Binary:** The absence of Sysmon EID 11 file creation events or process access events related to nmap.exe suggests the binary was not accessed or executed.

**Limited PowerShell Content:** Most PowerShell EID 4104 script block events contain only test framework boilerplate (Set-StrictMode calls) rather than actual scanning logic or error output that would indicate what happened with the nmap command.

## Assessment

This dataset provides excellent telemetry for detecting the *attempt* to execute network scanning tools via PowerShell, but limited visibility into the actual scanning activity. The PowerShell script block logging successfully captured the command execution, and the process creation events provide clear audit trails. However, the absence of nmap process execution and network activity significantly limits its utility for understanding the full attack lifecycle.

The data sources captured here (Security 4688, PowerShell 4104, Sysmon 1/7/10/11/17) are well-suited for detecting tool execution attempts and PowerShell-based reconnaissance commands. The missing network telemetry (Sysmon EID 3) would be crucial for detecting successful port scanning activity.

## Detection Opportunities Present in This Data

1. **PowerShell Script Block Detection:** Monitor PowerShell EID 4104 events for script blocks containing network scanning tools like "nmap", "portscan", "Test-NetConnection" with suspicious target parameters.

2. **Command Line Pattern Matching:** Detect Security EID 4688 process creation events where PowerShell command lines contain network reconnaissance tools ("nmap", "masscan", "zmap") combined with IP addresses or network ranges.

3. **PowerShell Process Spawning Patterns:** Alert on PowerShell processes (parent) spawning child PowerShell processes with command lines containing network scanning utilities, indicating potential reconnaissance activity.

4. **Network Tool Invocation in Script Blocks:** Create signatures for PowerShell script blocks that invoke external network scanning binaries, particularly when combined with localhost or internal network targets.

5. **Process Access Correlation:** Monitor for PowerShell processes accessing multiple child processes in rapid succession, which may indicate automated tool execution or process enumeration.

6. **Execution Policy Bypass with Network Commands:** Detect the combination of PowerShell execution policy bypass (Set-ExecutionPolicy) followed by script blocks containing network scanning commands within short time windows.
