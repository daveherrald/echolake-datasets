# T1547.014-2: Active Setup — Add Malicious StubPath to Existing HKLM Entry

## Technique Context

Active Setup (T1547.014) is a Windows mechanism designed to run per-user initialization tasks the first time a user logs on after a new component is installed. The system compares component version numbers stored in `HKLM\SOFTWARE\Microsoft\Active Setup\Installed Components\<GUID>` against per-user versions in HKCU, then executes the `StubPath` command for any component where the HKLM version is newer. Attackers abuse this by writing a malicious `StubPath` value to an existing Active Setup entry, causing arbitrary payload execution the next time any user logs on.

This test targets the existing `{C9E9A340-D1F1-11D0-821E-444553540600}` entry (Internet Explorer Core Fonts) and sets the `StubPath` to `C:\Windows\System32\calc.exe`. Rather than waiting for a real logon, the test immediately triggers execution via `runonce.exe /AlternateShellStartup`, which simulates the Active Setup processing that occurs at logon.

This dataset captures the **undefended** execution on ACME-WS06 with Defender disabled. The defended variant (ACME-WS02) produced 51 sysmon, 15 security, and 38 powershell events. The undefended dataset shows a dramatically higher 147 sysmon, 10 security, and 100 powershell events — the difference is almost entirely explained by the 111 Sysmon EID 11 (FileCreate) events. This disparity reflects the different profile setup state of ACME-WS06 compared to ACME-WS02: when `runonce.exe /AlternateShellStartup` executed, it triggered a first-logon profile initialization sequence for the SYSTEM account on WS06 that had not previously run, generating 111 file creation events as Windows built out the SYSTEM profile.

## What This Dataset Contains

The dataset spans approximately 15 seconds on ACME-WS06 and contains 257 events across three log sources.

**PowerShell EID 4104** captures the full attack script:

```powershell
Set-ItemProperty "HKLM:\SOFTWARE\Microsoft\Active Setup\Installed Components\{C9E9A340-D1F1-11D0-821E-444553540600}" "StubPath" "C:\Windows\System32\calc.exe" -Force
& $env:SYSTEMROOT\system32\runonce.exe /AlternateShellStartup
```

The test writes the StubPath value then immediately triggers the Active Setup processing chain.

**Sysmon (147 events, EIDs 1, 7, 10, 11, 13, 17):**

- **EID 1 (ProcessCreate):** Four events documenting the attack chain: `whoami.exe` (T1033, pre-check); `powershell.exe` (T1083) with the StubPath write + runonce invocation as the command line; a second `whoami.exe` for cleanup verification. The PowerShell command line captures the full two-step payload.

- **EID 11 (FileCreate):** 111 events — the bulk of this dataset. These are generated by the Active Setup processing of `runonce.exe /AlternateShellStartup` initializing the SYSTEM account's Windows shell profile. The file creates include:
  - `C:\Windows\System32\config\systemprofile\Links\Downloads.lnk` (tagged `T1187 Forced Authentication`)
  - `C:\Windows\System32\config\systemprofile\AppData\Roaming\Microsoft\Windows\SendTo\` and subfiles (tagged `T1574.010`)
  - `C:\Windows\System32\config\systemprofile\AppData\Local\Microsoft\Edge\` (tagged `T1574.010`)
  - And approximately 106 additional profile setup file creates

  These EID 11 events are not indicators of T1547.014 specifically — they reflect the consequence of `runonce.exe` triggering Active Setup and shell initialization for a first-time SYSTEM profile. They are part of the realistic environment noise produced by executing the technique as designed.

- **EID 13 (RegistrySetValue):** Six events. The attack-specific write is present: `Set-ItemProperty` on `HKLM:\SOFTWARE\Microsoft\Active Setup\Installed Components\{C9E9A340-D1F1-11D0-821E-444553540600}` with `StubPath = C:\Windows\System32\calc.exe`. Additional EID 13 events capture background Windows maintenance activity coinciding with the test window (Software Protection Platform task updates, etc.).

- **EID 10 (ProcessAccess):** Four events tagged `T1055.001` — test framework handle acquisition.

- **EID 17 (PipeCreate):** Two named pipe creation events.

- **EID 7 (ImageLoad):** 20 DLL load events.

**Security (10 events, all EID 4688):** Process creation records for `whoami.exe` (twice), the attack `powershell.exe`, `runonce.exe /AlternateShellStartup`, and the full Active Setup execution chain triggered by runonce:

```
NewProcessName: C:\Windows\System32\runonce.exe
CommandLine: "C:\Windows\system32\runonce.exe" /AlternateShellStartup

NewProcessName: C:\Windows\System32\unregmp2.exe
CommandLine: "C:\Windows\System32\unregmp2.exe" /FirstLogon

NewProcessName: C:\Windows\System32\ie4uinit.exe
CommandLine: "C:\Windows\System32\ie4uinit.exe" -UserConfig

NewProcessName: C:\Windows\System32\ie4uinit.exe
CommandLine: C:\Windows\System32\ie4uinit.exe -ClearIconCache

NewProcessName: C:\Windows\System32\unregmp2.exe
CommandLine: "C:\Windows\System32\unregmp2.exe" /FirstLogon

NewProcessName: ...\Edge\Application\146.0.3856.59\Installer\setup.exe
CommandLine: "...\setup.exe" --configure-user-settings --verbose-logging --system-level --msedge --channel=stable
```

The `runonce.exe /AlternateShellStartup` process and its chain of child processes are the direct result of the Active Setup mechanism executing. Notably, `calc.exe` (the injected StubPath payload) is **not** visible in the Security channel samples — either it ran and its process create event was not sampled, or the StubPath for that specific GUID was not re-processed in this run.

## What This Dataset Does Not Contain

**calc.exe process create is not confirmed in the sampled events.** The Security and Sysmon samples do not include a 4688 or EID 1 record for `calc.exe` launching from the Active Setup chain. The runonce execution triggered many other Active Setup components (`unregmp2.exe`, `ie4uinit.exe`, `Edge` installer), and the `calc.exe` StubPath may have been superseded or already processed in a prior run of the SYSTEM profile.

**No explicit T1547.014 Sysmon rule tag.** The EID 13 for the StubPath write fires on the default rule (no named technique tag), meaning sysmon-modular does not have a specific named rule for the Active Setup Installed Components path.

**No Security EID 4657.** Registry write auditing via SACL is not enabled.

## Assessment

The defining characteristic of this dataset compared to the defended variant is the 111-event EID 11 burst generated by `runonce.exe /AlternateShellStartup` initializing the SYSTEM profile. In the defended dataset this expansion did not occur (ACME-WS02's SYSTEM profile was already initialized), resulting in far fewer file creation events. This is an important calibration point: the same technique can produce dramatically different event volumes depending on the target system's prior state.

The core technique telemetry — the StubPath write in Sysmon EID 13, the `powershell.exe` process create in Sysmon EID 1 and Security EID 4688, and the `runonce.exe /AlternateShellStartup` execution — is present and actionable in both datasets regardless of the profile-initialization noise.

The Active Setup execution chain (`runonce.exe` spawning `unregmp2.exe`, `ie4uinit.exe`, Edge installer) is a direct, observable consequence of the technique's activation mechanism. This chain is unusually noisy for a persistence test — most T1547 tests simply make a configuration change and rely on a future trigger. Here the trigger was executed immediately, generating a visible execution chain.

## Detection Opportunities Present in This Data

- **Sysmon EID 13:** Writes to `HKLM\SOFTWARE\Microsoft\Active Setup\Installed Components\<GUID>\StubPath` by any process other than a legitimate installer. The `StubPath` value name in combination with the Active Setup path is highly specific.

- **Security EID 4688:** `powershell.exe` command lines referencing `Active Setup`, `Installed Components`, and `StubPath` together. The GUID path is also searchable — `{C9E9A340-D1F1-11D0-821E-444553540600}` is the IE Core Fonts component; writes to a StubPath under any GUID by non-Microsoft processes warrant review.

- **Security EID 4688:** `runonce.exe /AlternateShellStartup` spawned by an unusual parent (PowerShell, cmd.exe, or other non-system processes). Legitimate `runonce.exe /AlternateShellStartup` is typically invoked by `winlogon.exe` or `explorer.exe` during logon — not by scripting hosts.

- **PowerShell EID 4104:** Script blocks containing `Set-ItemProperty` targeting `HKLM:\SOFTWARE\Microsoft\Active Setup\Installed Components\` with `StubPath` as the Name parameter.

- **Process chain analysis:** `runonce.exe /AlternateShellStartup` followed by execution of processes that do not normally appear in this chain (anything other than `unregmp2.exe`, `ie4uinit.exe`, standard Windows setup components) indicates a malicious StubPath was processed.

- **Value baseline:** The `StubPath` values for existing Active Setup entries are stable. Any change to an existing entry's StubPath, or the addition of a new entry with a non-Microsoft StubPath, should be flagged.
