# T1614.001-7: System Language Discovery — Discover System Language with dism.exe

## Technique Context

T1614.001 (System Language Discovery) covers adversary enumeration of configured language and locale settings. The Deployment Image Servicing and Management tool (`dism.exe`) is a Windows inbox utility primarily used for image servicing, but it can also query currently installed language packs on a live system via `dism /online /get-intl`. The output includes the system locale, input locale, UI language, UI language fallback, and installed language packs. Because `dism.exe` is a signed Microsoft binary commonly used by IT administrators, its use for locale discovery can blend with legitimate administrative activity. The sysmon-modular configuration tags `dism.exe` invocations with `technique_id=T1548.002` (Bypass UAC) due to a known UAC bypass involving DISM, even when the command is benign.

## What This Dataset Contains

The dataset spans roughly 5 seconds across three log sources (27 Sysmon events, 18 Security events, 34 PowerShell events).

**Sysmon Event 1** (ProcessCreate) captures:
- `whoami.exe` — ART test framework identity check (tagged `technique_id=T1033`)
- `cmd.exe` — command shell used to invoke DISM (tagged `technique_id=T1059.003`)
- `dism.exe` — the language discovery command (tagged `technique_id=T1548.002`)

The sysmon-modular rule fires on `dism.exe` with the UAC-bypass tag, which is a false-positive label in this context but still correctly captures the process.

**Security Events** in this dataset include logon events not typically present in others:
- **4624**: Service logon (Type 5) for NT AUTHORITY\SYSTEM — DISM spawns a service host or worker process that generates a new logon session.
- **4627**: Group membership enumeration associated with the new logon.
- **4672**: Special privileges assigned to the SYSTEM logon — SeAssignPrimaryTokenPrivilege, SeTcbPrivilege, and other SYSTEM-level privileges.

These logon/privilege events are generated by DISM's internals and provide additional context that this tool was used in a privileged context.

**Security Event 4689** records exit of `Dism\DismHost.exe` — confirming that DISM spawned its host worker process as part of the `/get-intl` operation.

**Security Event 4688** records `cmd.exe`, `dism.exe`, and `DismHost.exe` process creation, all as SYSTEM.

**Sysmon Event 10** (ProcessAccess) fires on the PowerShell test framework accessing child processes.

**Sysmon Event 7** (ImageLoad) shows .NET and Defender DLL loads.

**PowerShell Events 4103/4104** contain only ART test framework. The actual DISM invocation is via `cmd.exe`.

## What This Dataset Does Not Contain

The output of `dism /online /get-intl` — the actual locale and language pack data — is not captured in any log source. DISM does not write its output to an auditable file in this test.

The Sysmon ProcessCreate filter's assignment of `technique_id=T1548.002` to `dism.exe` is a mislabeling for this specific use case. Analysts should be aware that this rule fires on all `dism.exe` invocations, not only UAC-bypass attempts. The actual UAC-bypass variant would involve a different command-line pattern.

## Assessment

This dataset is notable for the logon telemetry (4624, 4627, 4672) generated by DISM's internal service architecture, which is not present in most other tests in this batch. The logon events add context and make DISM-initiated activity more distinctive. The `technique_id=T1548.002` sysmon rule tag is worth understanding: it is a heuristic that matches `dism.exe` generally, not just in UAC-bypass contexts. Defenders should inspect the full command line to distinguish language discovery from actual bypass attempts. Defender was active and did not block this test.

## Detection Opportunities Present in This Data

- **Sysmon Event 1**: `dism.exe` spawned from `cmd.exe` → `powershell.exe` chain, especially as SYSTEM. The `CommandLine` field will contain `/online /get-intl` or similar locale-query arguments in the raw event.
- **Security Event 4688**: Full command line for `dism.exe` and `DismHost.exe`. The `/get-intl` argument is a clear indicator of locale discovery intent.
- **Security Events 4624/4627/4672**: Service logon and privilege assignment generated by DISM; the combination of these events co-occurring with `dism.exe` process creation in a short time window is distinctive.
- **Security Event 4689**: `DismHost.exe` exit event confirms DISM ran to completion (not aborted mid-execution), suggesting the locale query succeeded.
- **Sysmon-modular rule caution**: The `T1548.002` tag on `dism.exe` in Event 1 is a broad heuristic; triage should always examine the actual command line before escalating as a UAC-bypass alert.
