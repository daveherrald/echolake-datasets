# T1614.001-2: System Language Discovery — Discover System Language with chcp

## Technique Context

T1614.001 (System Language Discovery) covers adversary enumeration of system language and locale settings. The `chcp` (Change Code Page) command, when run without arguments, displays the currently active code page number for the console session. Code pages map to language/region settings: `437` is US English, `850` Western European, `932` Japanese, `1251` Cyrillic, `936` Simplified Chinese, and so on. Adversaries use `chcp` as a fast, low-footprint way to infer the system's primary language without querying the registry, making it a common component of automated reconnaissance scripts and malware dropper stagers.

## What This Dataset Contains

The dataset spans roughly 4 seconds across three log sources (16 Sysmon events, 12 Security events, 34 PowerShell events).

**Sysmon Event 1** (ProcessCreate) records:
- `whoami.exe` — ART test framework identity verification (tagged `technique_id=T1033`)
- `cmd.exe` — command shell used to invoke `chcp` (tagged `technique_id=T1059.003`)

**Security Event 4689** (process exit) records `chcp.com` completing — confirming that `chcp.com` was spawned as a child of `cmd.exe`. While the Security Event 4688 (process creation) for `chcp.com` is present in the raw security log, the bundled dataset includes the 4689 exit event which lists `C:\Windows\System32\chcp.com` as the process name, establishing that the command ran.

**Security Event 4688** records process creation events for `powershell.exe`, `whoami.exe`, `cmd.exe`, and `chcp.com`, all under NT AUTHORITY\SYSTEM (logon ID 0x3E7). Command-line auditing is enabled, so the full invocation of `cmd.exe /c chcp` (or equivalent) is present in the 4688 event for `cmd.exe`.

**Sysmon Event 10** (ProcessAccess) fires on the test framework accessing child processes, tagged `technique_id=T1055.001`.

**Sysmon Event 7** (ImageLoad) shows .NET and Defender DLL loads into PowerShell.

**PowerShell Events 4103/4104** contain only the ART test framework (`Set-ExecutionPolicy Bypass`). The `chcp` call runs through `cmd.exe` outside the PowerShell pipeline.

## What This Dataset Does Not Contain

`chcp.com` does not appear in Sysmon Event 1 because the include-mode ProcessCreate filter does not have a rule for `chcp.com` — it is not a commonly flagged LOLBin in the sysmon-modular configuration. Its execution is captured only via Security Event 4688/4689. The code page value returned by `chcp` (the language indicator) is not recorded by any log source.

No registry, network, or file activity is generated by `chcp`. The technique leaves an exceptionally small footprint — a single console command that reads a memory value and prints a number.

## Assessment

This dataset illustrates how a minimal language-check technique generates telemetry primarily in the Security event log rather than Sysmon. `chcp.com` executes as a child of `cmd.exe` and produces a 4688 event with command-line evidence, but Sysmon's include-mode ProcessCreate filter misses it. This is a useful reminder that Sysmon alone — even with a comprehensive configuration — does not capture all process creation events. The Security log's 4688 events, enabled with command-line auditing, provide coverage here. The code page result is not recoverable from logs.

## Detection Opportunities Present in This Data

- **Security Event 4688**: `chcp.com` spawned from `cmd.exe` as a child of PowerShell or another scripting host, especially as NT AUTHORITY\SYSTEM. The command line will be present if command-line auditing is enabled.
- **Security Event 4688 / Sysmon Event 1**: `cmd.exe` spawned from PowerShell or `wscript.exe`/`cscript.exe` with an argument string invoking `chcp` warrants attention when part of a reconnaissance sequence.
- **Behavioral correlation**: `chcp` followed by other language/locale discovery commands (`reg query NLS`, `GetLocale`, `getmac`, `ipconfig`, `systeminfo`) within a short window is a strong indicator of automated pre-attack reconnaissance.
- **Sysmon gap awareness**: Detection of `chcp.com` requires Security Event 4688 with command-line auditing. Relying solely on Sysmon ProcessCreate will miss this command in standard configurations.
