# T1018-4: Remote System Discovery — Remote System Discovery - ping sweep

## Technique Context

T1018 Remote System Discovery is a foundational discovery technique where adversaries attempt to identify systems within a network environment. This technique is crucial for lateral movement planning, as attackers need to understand the network topology and identify potential targets before expanding their foothold. The ping sweep variant specifically uses ICMP echo requests to systematically probe IP address ranges, making it one of the most common and straightforward methods for network reconnaissance.

Detection engineers focus heavily on this technique because it often represents early-stage reconnaissance activity that can indicate an active compromise. The technique generates distinctive patterns of network traffic and process execution that are relatively easy to detect when performed at scale. Community detection strategies typically focus on identifying rapid sequences of ping commands targeting sequential IP addresses, unusual process trees involving cmd.exe and ping.exe, and network traffic patterns showing systematic ICMP probes.

## What This Dataset Contains

This dataset captures a comprehensive ping sweep execution through PowerShell and cmd.exe. The primary command line executed was `"cmd.exe" /c for /l %i in (1,1,254) do ping -n 1 -w 100 192.168.1.%i`, which systematically pinged IP addresses 192.168.1.1 through 192.168.1.254.

The Security channel shows the complete process execution chain: PowerShell (PID 8128) spawned cmd.exe (PID 7544) with Security EID 4688 events, which then generated 254 individual PING.EXE processes. Each ping process used the command pattern `ping -n 1 -w 100 192.168.1.X` where X incremented from 1 to 254. The Security events show all ping processes exiting with status 0x1, indicating the target IPs were unreachable.

Sysmon captured 87 events including process creation events (EID 1) for whoami.exe, cmd.exe, and multiple PING.EXE instances. The Sysmon events are tagged with `technique_id=T1018,technique_name=Remote System Discovery` for the ping processes and `technique_id=T1059.003,technique_name=Windows Command Shell` for cmd.exe. Process access events (EID 10) show PowerShell accessing the spawned processes with full access rights (0x1FFFFF).

PowerShell logging contains primarily test framework boilerplate with Set-ExecutionPolicy and Set-StrictMode commands, plus CIM module loading events. The ping sweep execution itself doesn't appear in the PowerShell script block logs, indicating it was launched via process execution rather than PowerShell cmdlets.

## What This Dataset Does Not Contain

The dataset lacks network-level telemetry showing the actual ICMP packets generated by the ping sweep. Windows Defender was active but did not block this reconnaissance activity, as ping sweeps using legitimate Windows tools typically don't trigger endpoint protection. The dataset shows no successful ping responses, indicating either the target network range was unused or ICMP was blocked.

DNS resolution events are missing, which would normally accompany network discovery if the attacker attempted to resolve hostnames. The Sysmon configuration's include-mode filtering for ProcessCreate means we only see ping.exe creation because it matched the T1018 detection rule, but we might be missing other supporting processes that didn't match known-suspicious patterns.

The dataset truncates before showing the complete ping sweep execution, containing only the first 39 ping attempts out of the expected 254. This limits analysis of the full technique execution pattern and timing characteristics.

## Assessment

This dataset provides excellent telemetry for detecting ping sweep reconnaissance through process monitoring. The Security channel's process creation and termination events offer complete visibility into the attack pattern, showing clear parent-child relationships and command-line arguments that would enable robust detection rules. The Sysmon data adds valuable context with process GUIDs, file hashes, and process access events.

The systematic nature of the ping execution creates highly detectable patterns: sequential process creation timestamps, identical command-line patterns with incremental IP addresses, and consistent exit codes. The combination of Security EID 4688 events and Sysmon EID 1 events provides redundant coverage that would be difficult for attackers to evade.

However, the dataset's value is somewhat limited by the truncated execution and lack of network telemetry. For comprehensive ping sweep detection, network flow data or firewall logs showing the actual ICMP traffic would strengthen the detection capability and provide additional correlation opportunities.

## Detection Opportunities Present in This Data

1. **Rapid Sequential Process Creation**: Multiple ping.exe processes created within seconds, each with incrementing IP addresses in command lines, detectable through Security EID 4688 or Sysmon EID 1 events

2. **Command Line Pattern Matching**: Detection of cmd.exe executing for loops with ping commands targeting IP ranges, specifically the pattern `for /l %i in (1,1,254) do ping` in process command lines

3. **Process Tree Analysis**: PowerShell spawning cmd.exe which rapidly creates multiple ping.exe children, creating a distinctive process hierarchy for lateral movement reconnaissance

4. **High-Frequency Network Discovery**: Multiple ping processes with identical parameters (-n 1 -w 100) targeting sequential IP addresses within a short timeframe, detectable through process creation frequency analysis

5. **Systematic Target Enumeration**: Sequential IP address targeting (192.168.1.1, 192.168.1.2, etc.) visible in command-line arguments, indicating automated network reconnaissance rather than legitimate troubleshooting
