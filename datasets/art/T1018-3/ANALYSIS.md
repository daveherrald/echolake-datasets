# T1018-3: Remote System Discovery — nltest

## Technique Context

T1018 Remote System Discovery involves adversaries attempting to get a listing of other systems by IP address, hostname, or other logical identifier on a network that may be used for Lateral Movement from the current system. The nltest.exe utility is a legitimate Windows domain administration tool that attackers commonly abuse for Active Directory reconnaissance. Specifically, `nltest /dclist` enumerates domain controllers in the current or specified domain, providing valuable intelligence for attackers planning lateral movement or privilege escalation within domain environments.

Detection engineers focus on monitoring execution of nltest.exe with domain enumeration parameters, especially when executed from non-administrative contexts or unusual parent processes. The technique is particularly concerning when observed alongside other discovery activities or from compromised endpoints.

## What This Dataset Contains

The dataset captures a clean execution of nltest domain controller enumeration via PowerShell. The process chain shows:

1. **PowerShell parent process** (PID 4768): `powershell.exe` running as NT AUTHORITY\SYSTEM
2. **Command shell intermediary** (PID 7812): `"cmd.exe" /c nltest.exe /dclist:%userdnsdomain%` 
3. **Target nltest execution** (PID 4788): `nltest.exe /dclist:%userdnsdomain%`

Key telemetry includes Security 4688 events capturing the full command line `nltest.exe /dclist:%userdnsdomain%` and exit status 0x0 indicating successful execution. Sysmon EID 1 events capture the cmd.exe creation with the full command line showing the environment variable expansion. The dataset also includes a separate whoami.exe execution (PID 1160) showing additional system reconnaissance activity.

Process access events (Sysmon EID 10) show PowerShell accessing both the whoami and cmd processes with full access rights (0x1FFFFF), indicating normal parent-child process relationships during execution.

## What This Dataset Does Not Contain

The dataset lacks the actual nltest.exe process creation in Sysmon logs, likely filtered out by the include-mode ProcessCreate configuration that focuses on known suspicious patterns. While nltest.exe is sometimes flagged as a LOLBin, it may not be covered by this particular sysmon-modular configuration. The Security 4688 event does capture the nltest execution, providing the critical command line evidence.

No network connections or DNS queries are captured, which would normally show the domain controller enumeration traffic generated by nltest. The dataset also doesn't contain any file system artifacts that nltest might create or access during domain enumeration.

## Assessment

This dataset provides solid coverage of nltest-based domain discovery through Security audit logs, with clear command line evidence of the technique execution. The Security 4688 events offer reliable detection opportunities with full command line capture. However, the filtering of nltest.exe from Sysmon ProcessCreate events reduces the depth of process-level telemetry available.

The combination of PowerShell and cmd.exe process creation events provides good context for understanding the execution chain, even without the final nltest process creation. The clean exit status confirms successful technique execution, making this useful for testing detection rules that rely on Security audit logs.

## Detection Opportunities Present in This Data

1. **nltest domain enumeration command line** - Security 4688 events showing `nltest.exe /dclist` with domain parameters, particularly when executed via command shell intermediaries

2. **PowerShell spawning reconnaissance tools** - Process creation patterns showing powershell.exe spawning cmd.exe with nltest execution, indicating scripted domain discovery

3. **Environment variable expansion in discovery commands** - Command lines containing `%userdnsdomain%` expansion suggesting automated domain enumeration techniques

4. **Process access patterns during reconnaissance** - Sysmon EID 10 events showing PowerShell accessing spawned reconnaissance processes with full access rights

5. **Multiple reconnaissance tool execution** - Correlation of nltest execution with other discovery tools like whoami.exe within short time windows

6. **System-level reconnaissance activity** - Discovery commands executed under SYSTEM context, potentially indicating post-exploitation or malware activity
