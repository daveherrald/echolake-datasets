# T1003.005-1: Cached Domain Credentials — Cached Credential Dump via Cmdkey

## Technique Context

T1003.005 Cached Domain Credentials focuses on adversaries attempting to extract cached domain logon information stored locally on Windows systems. When users authenticate to Active Directory from domain-joined machines, Windows caches credential hashes locally in the LSA Secrets and Security Account Manager (SAM) to allow logon when domain controllers are unavailable. Attackers target these cached credentials to perform lateral movement or credential harvesting attacks.

This specific test (T1003.005-1) uses the legitimate Windows `cmdkey.exe` utility to enumerate stored credentials. While `cmdkey` primarily manages generic credentials rather than cached domain credentials, it represents a common reconnaissance technique attackers use to discover what credential stores are available on compromised systems. The detection community focuses on monitoring credential manager access patterns, unusual use of credential enumeration tools, and process execution chains that indicate credential harvesting attempts.

## What This Dataset Contains

This dataset captures a straightforward execution of `cmdkey /list` through PowerShell. The key telemetry includes:

**Process execution chain**: Security event 4688 shows the full command-line progression: `powershell.exe` → `"cmd.exe" /c cmdkey /list` → `cmdkey  /list`. The intermediate `cmd.exe` process (PID 6428) executes with the command line `"cmd.exe" /c cmdkey /list`, which then spawns `cmdkey.exe` (PID 6944) with arguments `cmdkey  /list`.

**Sysmon process creation events**: EID 1 captures the creation of `whoami.exe` (PID 6268), `cmd.exe` (PID 6428), and `cmdkey.exe` (PID 6944) with full command-line arguments and process relationships. The `cmdkey.exe` process is correctly tagged with RuleName `technique_id=T1087,technique_name=Account Discovery` in the sysmon-modular configuration.

**Process access events**: Sysmon EID 10 shows PowerShell accessing both the `whoami.exe` and `cmd.exe` child processes with `GrantedAccess: 0x1FFFFF`, indicating full process access rights for process management.

**PowerShell telemetry**: The PowerShell channel contains only boilerplate test framework activities (`Set-ExecutionPolicy Bypass`) with no script block content related to the actual technique execution.

All processes execute under `NT AUTHORITY\SYSTEM` context and complete successfully with exit status `0x0`.

## What This Dataset Does Not Contain

This dataset lacks several important elements that would typically accompany credential harvesting attempts:

**Actual credential access**: The `cmdkey /list` command only enumerates stored credentials without accessing their contents. No events show actual credential extraction or LSASS process access.

**DPAPI activity**: Since `cmdkey` manages user-specific credential stores protected by DPAPI, there are no corresponding DPAPI decryption events that would indicate successful credential extraction.

**Registry access patterns**: Missing are registry events showing access to credential storage locations in HKEY_CURRENT_USER or HKEY_LOCAL_MACHINE credential manager keys.

**Network activity**: No network connections are generated by this technique, as it only performs local enumeration.

**File system artifacts**: The technique doesn't create credential dump files or other persistence mechanisms that would generate file creation events.

## Assessment

This dataset provides solid process execution telemetry for detecting `cmdkey` enumeration but represents only the reconnaissance phase of credential harvesting. The telemetry quality is excellent for building detections around suspicious use of credential management utilities, particularly when executed through PowerShell or command shells in automated fashion.

The process genealogy is clearly captured across both Security 4688 and Sysmon EID 1 events, providing redundant coverage for detection engineering. However, the dataset's utility is limited for understanding the complete credential harvesting kill chain since it only covers enumeration rather than extraction or abuse phases.

For building comprehensive T1003.005 detections, this dataset would need to be combined with examples showing actual cached credential extraction techniques like `reg save` operations targeting credential stores, LSASS dumping, or tools like Mimikatz accessing LSA Secrets.

## Detection Opportunities Present in This Data

1. **Cmdkey process execution monitoring** - Alert on `cmdkey.exe` execution with `/list` argument, especially when spawned by scripting engines or command shells rather than interactive user sessions

2. **Command shell credential enumeration chains** - Detect process chains where `cmd.exe` or `powershell.exe` spawn credential management utilities like `cmdkey.exe`, particularly with enumeration flags

3. **System context credential enumeration** - Flag credential enumeration tools executing under `NT AUTHORITY\SYSTEM` or other privileged service accounts rather than standard user contexts

4. **Automated PowerShell credential discovery** - Monitor PowerShell processes that spawn multiple reconnaissance tools in sequence, including `whoami.exe` followed by credential enumeration utilities

5. **Process access patterns during credential operations** - Alert on processes accessing credential management utilities with full access rights (0x1FFFFF) which may indicate process injection or manipulation attempts

6. **Suspicious parent-child relationships** - Detect when credential enumeration occurs through indirect execution methods (`cmd.exe /c cmdkey`) rather than direct invocation, suggesting scripted or automated execution
