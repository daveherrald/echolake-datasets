# T1059.001-2: PowerShell — Run BloodHound from Local Disk

## Technique Context

T1059.001 (PowerShell) is the execution vehicle here; the payload is BloodHound, specifically the SharpHound ingestor loaded as a PowerShell module from a local file on disk. BloodHound is the standard tool for Active Directory attack path analysis. It queries the domain for users, groups, computers, ACLs, and trust relationships, then builds a graph that reveals paths from a compromised account to Domain Admin. `Invoke-BloodHound` (the PowerShell wrapper) is one of two common invocation methods alongside the standalone SharpHound.exe binary.

Running BloodHound generates distinctive telemetry: a PowerShell process importing a module from an unusual path, followed by WMI queries (via `WmiPrvSE.exe`), enumeration of local group memberships (Security EID 4798/4799), and LDAP traffic to the domain controller. The local-disk variant (this test) is distinguished from the download-cradle variant (test 3) in that the file is already present at `C:\AtomicRedTeam\atomics\..\ExternalPayloads\SharpHound.ps1` — no network download is required for the module itself, though the tool will still make LDAP connections to the DC.

In defended environments, Defender flags SharpHound.ps1 and terminates the process with exit code `0x80131501` before `Invoke-BloodHound` can run. This dataset captures the undefended execution, where SharpHound loads and runs.

## What This Dataset Contains

Security EID 4688 captures the key process creation for the PowerShell child executing the test (PID 0xd08, parent 0x1834):

```
"powershell.exe" & {import-module "C:\AtomicRedTeam\atomics\..\ExternalPayloads\SharpHound.ps1"
try { Invoke-BloodHound -OutputDirectory $env:Temp }
catch { $_; exit $_.Exception.HResult}
Start-Sleep 5}
```

The `import-module` target path traverses up from the atomics directory using `..` — a path pattern worth noting as it would appear unusual in process creation logs. A cleanup step is also captured: `"powershell.exe" & {Remove-Item $env:Temp\*BloodHound.zip -Force}`, showing the test deletes BloodHound output files from `%TEMP%` after the run.

Security EID 4688 also shows `WmiPrvSE.exe` being created (PID 0xf54) by `svchost.exe` — the WMI provider host spinning up to service queries generated by SharpHound's domain enumeration.

The Security channel's most significant contribution is the 5 EID 4798 events. The 5 EID 4798s are attributed to `WmiPrvSE.exe` (PID 0x1298) enumerating local group memberships for individual accounts: Administrator, DefaultAccount, Guest, mm11711, and WDAGUtilityAccount — this is BloodHound's WMI-based user enumeration.

Sysmon provides 36 events across EIDs 7, 1, 10, 17, and 11. EID 1 captures three process creates: `WmiPrvSE.exe` (`C:\Windows\System32\wbem\wmiprvse.exe -Embedding`, User: NT AUTHORITY\NETWORK SERVICE), `whoami.exe`, and the PowerShell process running the BloodHound command. EID 10 shows full-access handle opens (0x1FFFFF) from PowerShell to `whoami.exe` and to the child PowerShell process. EID 17 shows two named pipes for the two PowerShell instances: `\PSHost.134180039775341549.6196.DefaultAppDomain.powershell` and `\PSHost.134180039866850634.3416.DefaultAppDomain.powershell`.

The PowerShell channel has 103 EID 4104 events — all 103, with none of the 4100/4103 mix seen in other tests. This indicates the module loaded cleanly and all script blocks were logged. The sampled blocks include the test framework overhead and cleanup, but the full 103-event set contains SharpHound's function bodies.

Compared to the defended version (47 sysmon, 10 security, 50 powershell), the undefended version has fewer total events (36 sysmon, 29 security, 103 powershell), but the security channel is more informative: the 4798s documenting WMI-based user account enumeration are present.

## What This Dataset Does Not Contain

No Sysmon EID 3 network connection events are captured. SharpHound would make LDAP/LDAPS connections to the domain controller to enumerate AD objects, but the sysmon-modular configuration does not capture those connections. No DNS query events (EID 22) are present. The LDAP activity is therefore not directly visible.

No Sysmon EID 11 file creation events appear despite the cleanup step deleting `*BloodHound.zip` from `%TEMP%`. The zip file BloodHound creates during collection is not recorded being written, only deleted via the cleanup command line in EID 4688.

The 4799 events from `cribl.exe` are background instrumentation noise rather than SharpHound activity, which could complicate automated correlation. An analyst attributing all group enumeration in this window to BloodHound would be incorrect — the Cribl edge agent enumerates local groups during its operation on this host.

## Assessment

This dataset is valuable for modeling the WMI-mediated user and group enumeration that SharpHound performs during AD collection. The distinction between the `cribl.exe`-sourced EID 4799s (background) and the `WmiPrvSE.exe`-sourced EID 4798s (SharpHound) illustrates a common challenge: the same event types appear from both malicious and legitimate background processes. The Security 4688 command line with the `import-module` path through `ExternalPayloads\SharpHound.ps1` and the explicit `Invoke-BloodHound -OutputDirectory $env:Temp` is strong detection material.

## Detection Opportunities Present in This Data

1. EID 4688 with `import-module` targeting a path containing `SharpHound.ps1` — the filename itself is a known-bad indicator regardless of path.
2. EID 4688 with `Invoke-BloodHound -OutputDirectory $env:Temp` — a specific function name and output destination associated with SharpHound's collection phase.
3. EID 4688 cleanup step `Remove-Item $env:Temp\*BloodHound.zip -Force` — output cleanup matching BloodHound's default zip naming pattern in `%TEMP%`.
4. Security EID 4798 (`WmiPrvSE.exe` as the enumerating process) for multiple local accounts in rapid succession — bulk per-user group membership queries via WMI within a short time window.
5. Sysmon EID 1 for `WmiPrvSE.exe` with `CommandLine: wmiprvse.exe -Embedding` and `User: NT AUTHORITY\NETWORK SERVICE` spawning concurrently with PowerShell activity — the network-service WMI provider activating during a PowerShell session.
6. Sysmon EID 10 with `GrantedAccess: 0x1FFFFF` from PowerShell to a child PowerShell process — the test framework process accessing the test execution process with full rights.
7. EID 4688 path traversal pattern `C:\AtomicRedTeam\atomics\..\ExternalPayloads\` — a relative path component in an `import-module` argument indicating staged external payloads.
